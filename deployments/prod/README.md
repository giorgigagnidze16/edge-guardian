# EdgeGuardian — Production on GCP

Minimal operator runbook. After the one-time setup here, deploys are automated
by GitHub Actions and agents self-update; ongoing involvement is ~zero.

## One-time setup (~10 min)

### 1. GCP project + auth (your account)
```bash
gcloud auth login
gcloud auth application-default login
gcloud config set project <YOUR_PROJECT_ID>   # billing must be enabled
```

### 2. Fill Terraform inputs
```bash
cd deployments/terraform
cp terraform.tfvars.example terraform.tfvars
# edit: project_id, region/zone, github_repo (owner/repo)
```

### 3. Bring up infra + cluster prerequisites (one script)
```bash
LETSENCRYPT_EMAIL=you@example.com ./deployments/prod/bootstrap.sh
```
This runs `terraform apply` (GKE cluster, Artifact Registry, 2 static IPs,
Workload Identity Federation) and installs ingress-nginx (on the static IP),
cert-manager, and the Let's Encrypt ClusterIssuer. It prints the Terraform
outputs at the end — keep them.

### 4. Fill `values-prod.yaml` from the outputs
Set in `deployments/helm/edgeguardian/values-prod.yaml`:
- `ingress.baseDomain` = `<base_domain>` output (e.g. `34-1-2-3.sslip.io`)
- `controller.agentInstaller.controllerUrl` = `https://controller.<base_domain>`
- `controller.agentInstaller.brokerUrl` = `tcp://<mqtt_host>:1883`
- `controller.agentInstaller.mtlsBrokerUrl` = `ssl://<mqtt_host>:8883`
- `emqx.external.loadBalancerIP` = `<mqtt_ip>` output

### 5. Create the secrets file (local, gitignored)
```bash
cp deployments/helm/edgeguardian/values-prod-secrets.yaml.example \
   deployments/helm/edgeguardian/values-prod-secrets.yaml
# fill each REPLACE_ME; generate strong values with: openssl rand -base64 32
```

### 6. Add GitHub repo secrets (for the deploy pipeline)
See the values printed by bootstrap. Set these as GitHub Actions secrets
(`gh secret set NAME`): `GCP_PROJECT`, `GCP_REGION`, `GKE_CLUSTER`, `GKE_ZONE`,
`WIF_PROVIDER` (= `wif_provider` output), `DEPLOY_SA` (= `deployer_sa_email`
output), plus the app secrets from `values-prod-secrets.yaml`
(`PG_PASSWORD`, `KEYCLOAK_ADMIN_PASSWORD`, `MINIO_PASSWORD`, `GRAFANA_PASSWORD`,
`EMQX_DASHBOARD_PASSWORD`, `EMQX_CONTROLLER_PASSWORD`, `EMQX_BOOTSTRAP_PASSWORD`,
`NEXTAUTH_SECRET`, `KEYCLOAK_CLIENT_SECRET`, `CA_ENCRYPTION_KEY`,
`OTA_SIGNING_KEY`). The exact list + a `gh secret set` block is generated by
`scripts/gh-secrets.sh` (Phase B).

### 7. First deploy
Either push to `main` (the deploy workflow builds images and installs the
chart), or do it once manually:
```bash
helm upgrade --install edgeguardian deployments/helm/edgeguardian \
  -n edgeguardian --create-namespace \
  -f deployments/helm/edgeguardian/values-prod.yaml \
  -f deployments/helm/edgeguardian/values-prod-secrets.yaml \
  --wait --timeout 20m
```

### 8. First-org bootstrap
Open `https://ui.<base_domain>`, sign in via Keycloak (register). This creates
the first org + CA and unblocks EMQX's init container.

## After setup

- **Deploys:** every push to `main` builds + deploys automatically.
- **Agent updates:** tag `agent-vX.Y.Z` to publish a new agent version; devices
  with auto-update on converge on their own.
- **Teardown:** `terraform -chdir=deployments/terraform destroy` (plus
  `helm uninstall edgeguardian -n edgeguardian` first to release the LBs).
